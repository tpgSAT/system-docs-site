#   idとJavaScriptについて
　以前、`<a href="#">`の＃の部分にリンク先を記述しました。

　実際のHP運営では、リンク先のページも自分で作成したページであることが多く、https~というリンクを書くのではなくリンク先のページのindex.htmlへのパスを記述することがほとんどです。


　前回は、別のページへのリンクではなく、ボタンを押すと同じページ内で指定の場所までスクロールするという動きを作る必要がありました。そこで使うのがidというものです。


　idはクラスと同じようにタグに対して付与できます。クラスと同じ名称を使っても構いません。例えば、次のようになります。
```html
<div class="ex test" id="ex">
    <!-- exとtestというクラス名を付与、exというidを付与 -->
</div>
```
　ここで、これまでリンクやパスを入力していた部分に`#id名`を入力すると、そのidを持つ要素の部分までページが飛びます。

　実際に[目標物のページ](https://tpgd.jp/whatsnew/20230621_takifes/index.html)でDrinkというボタンを押すと、Drinkの部分にとび、リンクの末尾がindex.htmlからindex.html#drinkに変わります。


　CSSでこれまでタグ名やクラス名をプロパティとしてきましたが、id名を使うこともできます。しかし、管理が煩雑になるため原則避けることを推奨します。


　idの使い道はもう一つあり、それがJavaScriptです。

　JavaScriptではCSSでの実装ができない、あるいは複雑になってしまう表示をわりかし簡単に実装することができます。動きをつけるためにつかうんだなーと思ってもらえればOKです。

　JavaScriptの説明は気が向いたときに[こちら](#)を参照してもらえたらと思いますが、CSSでクラス名を使っていたように、JavaScriptではid名で指定すると思っていてください。もちろんJavaScriptでもクラス名で扱うことはできますし、TPGの公式HPでもクラス名でやってしまっている箇所は多いと思います。

　一応、JavaScriptの指示はid名の方が処理が速いらしいですが、JavaScriptで複雑なコードを書いているわけではないのでまあ誤差です。


　それでもidを付与するメリットは、HTMLだけを見たときにここにJavaScriptが効いているかもしれないと気づけるところにあります。

　今後TPGのHPを運営する上で全く知らない人が書いたコードを元に編集したり、自分が書いたコードが知らない代に引き継がれたりすることを考えて、その方が親切かなーと思っています。
