#   一括処理
　前回aタグに一括処理をしたように、例えば本文や見出しのフォントサイズ、文字色などを一括指定しておくことは便利です。いちいち書く行が減る上、もし違う色やサイズなどを適用したい箇所があったらクラス名を割り当て、CSSをそのクラスに対して書いてあげれば良いだけです。

　一方で、一括処理は闇雲に適用すればいいものでもありません。例えば見出しの文字サイズが何種類もあるときに一括処理の指示を書いても、結局各クラスに対して上書きする必要がありあまり意味がありません。


　また、TPGの公式HPではほぼ全ページに適用する処理だけをまとめたCSSファイルがあり、ここでは要素の表示幅を画面幅の70％にする処理を書いています。しかし、イベントのページやHoyaのページでは表示幅をもう少し広げたいことが多くこのCSSを読み込んではいけなかったり、共通の一括指示が効いていることに気づかず思うように処理がうまくいかない、となってしまうこともあります。TPGのページを運用する際にはこうした一括指示が現状どうなっているかについてまとめたページを用意しておくので、原因のわからない詰まりがあれば参照してください。


　ここでは、pタグとh3タグ(特に理由はないですが、見出しはh3タグを使うようにしています)に適用する一括処理と、ページ全体のフォントをデフォルトから変更する処理を書きます。

　今回書くCSSは次のようになります。
```css
html {
    scroll-behavior: smooth;/* ページ内のボタンを押したときにスクロール風の動きをする */
    font-size: 1.2vw;/* rem単位の基準値 */
}
body { 
    margin: 0;/* 余計な余白が勝手につくことがあるので、それを制限 */
}
header {
    font-family: 'Open Sans', sans-serif;/* ヘッダーのフォント設定 */
}
p, li{
    font-size: 1rem;
    color: #181818;
}
h3 {
    font-size: 1.5rem;
}
```
　フォントの種類の指定は、font-familyで行います。TPGのページでは、無料で使えるWebフォント(パソコンにフォントがインストールされているか否かによらず誰に対しても表示できるフォント)であるgoogle fontを使っています。google fontの使い方は、[google fontのサイト](https://fonts.google.com/?preview.text_type=paragraph&stylecount=1)にアクセスし、好きなフォントをクリックし使いたい文字の太さを選んで画面右の青い＋ボタンを押します。画面右側にコピペするコードが出てくるので、それをHTMLとCSSにそれぞれ貼り付けます。

TPGのページのヘッダーで使っているフォントはOpen Sansで、本文はIBMPlexSansJP、なにかが起きてこのフォントが読み込めないときの予備としてsans-serifを設定しています。font-familyの値に設定した順に優先してフォントが使われます。フォントを読み込むためにHTMLのheadタグ内にに次のコードを追加してください。
```html
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans+JP:wght@300;400;700&family=Open+Sans:wght@300;400;700&display=swap" rel="stylesheet">
```
　リンク内に300;400;700とありますが、これは文字の太さとして300,400,700をインストールしています。どうも文字の太さをきれいに変更するには400と700をいれておくと良いみたいです。ちなみに、文字の太さは`font-weight: bold;`のようにfont-weightで変えます。
