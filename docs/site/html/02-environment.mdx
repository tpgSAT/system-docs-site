# 作業環境の準備

まずは、コードを書く環境を整えましょう。VisualStudioCode(以下VScode)というソフトを入れます。すでに環境を持っている人はそれを使ってもらって大丈夫です(今後の説明はVScodeに対してしか行いません)。

公式の[こちらのサイト](https://code.visualstudio.com/)にアクセスし、DownloadforWindowsというボタンをクリックします。「VSCodeUserSetup-x64-1.78.2.exe」のようなファイル名のものがダウンロードされたと思います。このファイルをダブルクリックすると、インストールのガイドが出ると思うので指示に従ってください。

VScodeのインストールが完了したら、今後使っていく上で便利なプラグインを入れます。必須ではないですが、なるべく入れておくといいです。

VScodeを開き、画面左にある四角いボタンをクリックします。

次に検索欄に、導入したいプラグインのキーワードを入力します。

導入したいプラグインをクリックし、インスト－ルボタンを押せばインストール完了です。


では、作業する上で便利なプラグインを紹介します。

#### Japanese Language Pack for Visual Studio Code

まず日本語化したい人はこれで日本語化してください。


#### Auto Close Tag

　HTMLでは、タグというものを用います。タグには、初めに書く開始タグと終わりに書く閉じタグというものがあります。開始タグを書くと自動で対応する閉じタグをつけてくれるのがこのプラグインです。


#### Auto Rename Tag

　これは、開始タグや閉じタグのどちらか一方を変更すると自動で対応するタグも変更してくれるものです。


#### Color Hightlight

　文字色や背景の色などを指定するときに、カラーコードで記入するのですが、カラーコードだとどんな色かわかりづらいものもあります。そのため、カラーコードにそのカラーで蛍光ペンを惹いたような装飾を施してくれるプラグインです。


#### HTML CSS Support

　ある箇所に設定した名前を別の場所で呼び出すということが頻繁にあるのですが、その呼び出しをサポートしてくれるものです。とりあえず今は便利な入力補助なんだなーと思っててもらえればいいです。


#### Prettier

インデントなどを自動で整えてくれます。これを使わないと、GitHubに共有した際にインデントなどの些細な違いが差分として扱われてしまいめんどくさいことになるので、これだけは全員入れてください。


HTML/CSSを書く環境は整いました。他にも今後使う可能性のあるプラグインはあるのですが、それは必要になったタイミングでまた説明します。
