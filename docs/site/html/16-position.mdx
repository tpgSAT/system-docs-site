#   絶対位置・相対位置の話
　[目標物のページ](https://tpgd.jp/whatsnew/20230621_takifes/index.html#drink)を開くと、Taki Fesのロゴや背景に写真があります。

　この部分をつくるには、要素を重ねる必要があります。そこで使うのが相対位置という概念です。何もしなければ要素は重ならず並んでいきますが、ここでは背景画像にロゴの画像と開催日時の文字を重ねる必要があります。


　使うのはCSSのpositionプロパティです。何も書かなければデフォルトはstaticです。

　positionの値として、static、relative、absolute、fixedの４種類があります。今やりたいのは背景画像を基準としてロゴや文字を重ねることなので、relativeとabsoluteを使います。

　位置指定の基準となる要素に`position: relative;`、その要素からの相対位置を指定したい要素に`position: absolute;`を設定します。ここで気をつけるのは、relativeは親要素に、absoluteはその子要素に設定する必要があるということです。


　つまり、divを一つ作成し、その中に背景画像・ロゴ・文字を置いて、背景画像に`position: relative;`、ロゴと文字に`position: absolute;`を設定しても、背景画像に対してロゴと文字は親子関係ではなく同じdivの同列な子要素なため思うような位置指定は効きません。

画像はimgタグで挿入するため、これを親要素とすることはできません。なので、適当なクラス名をつけたdivを親要素とし`position: relative;`、背景画像・ロゴ・文字に子要素として`position: absolute;`を設定します。そしてdivのサイズ(`width`と`height`)を仕様書の通りに設定、そして背景画像をそのdivのサイズいっぱいに配置します。


　位置指定として、[単位について](/w/w01_intro/006.5/README.md)でやったように親要素の幅を100％とするパーセント指定が使えます。基本的には要素の左上角を原点として右向き/下向きに軸を取り、どの方向(top,right,bottom,left)にどの程度ずらすかを`top: 90px;`などのように書きます。

　場所を指定する際に、例えば幅を60％にした画像を中央に置きたいなら`left`(position: relative/absolute;を使っていないならmargin-left)を20％にすれば良いですが、幅がわからないテキスト要素を中央に置きたい場合、相対位置/絶対位置指定ではtop/leftをいくつにすれば良いかわかりませんし、画像やdiv要素でも横幅を変更したらleftも変更しなければならず不便です。そこで、要素の中央を移動する`transform`というプロパティを使います。テキスト要素では座標指定の基準点は上端・横幅中央にあり、画像やdiv要素では基準点は上左角にあります。topやleftで指定されるのは、親要素の基準点(左上角)から要素の基準点の距離です。そこで、子要素の基準点を上下左右中央に移動してしまえば良いのです。
```css
transform: translate(-50%, -50%);
```
　これは、変更後の基準点から見て元の基準点がx軸・y軸方向にそれぞれ要素の横幅・縦幅の-50%ずれた位置にあるということです。x軸・y軸はそれぞれ右向き・下向きに伸びます。


　また、要素を重ねるということはどの要素が上の層になるか管理しなければなりません。これは`z-index: 10;`のようにします。10の部分には任意の整数値を指定できますが、適当な数字を使うと管理しづらくなるためTPGの公式HPでは0~10を原則としています。数字が大きい方が上です。
