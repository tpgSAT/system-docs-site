# 閑話　サイズの単位について

HP を作る上で、文字や画像サイズの指定など、サイズ指定は必須です。ここで、メジャーな単位を紹介しておきます。

　自分でページをつくるだけなら好きなように作れば良いのですが、TPG の公式 HP を運営するとなるとサイト全体の統一なども必要な上、記述方法がバラバラだとそのコードをテンプレとして再利用したり、多少の修正を加えるのがめんどくさいです。

　逆に、ある程度の基準を定めてある CSS ファイルが存在するので、それを元にあとは好きなように設定してもらえたらと思います。


**px(ピクセル)**：よく出てくる単位です。cm のような、不変な単位です。例えば、TPG の公式 HP のヘッダーの高さは 90px です。


**vw**：viewport width の略です。100vw がブラウザの表示幅なので、ブラウザの表示幅のどのくらいの比率を占めるかを表します。フォントサイズを vw 単位で定義していれば、ブラウザの表示幅を多少変えても見出しが意図しないところで折り返されたり、枠をはみ出してしまうことが起きづらくなります。

　一方、一般に画面が小さいスマホほど文字が大きい方が良いので、この場合は不向きとも言えます。実際 TPG の公式 HP では、PC 版表示における文字サイズは vw で指定し、---スマホ版では px で指定しています。--- スマホ版も vw を使うことにします。2023.4 現在はまだ px のはずです。


**vh**：viewport height の略です。vw は画面幅を 100 としたときの比率だったのに対し、vh は画面の高さ(縦)を 100 としたときの比率を表します。

　使いどころがいまいち微妙ですが、例えばスマホで画面の何割をこの画像で占めたい、などと制限のあるデザインのときには使えるかもしれません。私が思うメリットは、デザインのイメージを反映しやすいことです。たまーに使います。


**rem**：これはある基準値が定められているときに、その基準に対する比率を表す単位です。例えば 1.2vw を基準としている CSS ファイル内で、ある p タグに 1rem という単位をつけたらこの p タグのフォントサイズは 1.2vw です。

　 TPG の公式 HP の PC 版では、基準が 1.2vw、a タグと p タグは 1rem、h3 タグは 2.5rem と記述された全ページに共通の CSS を適用しています。部分的に変えたい場合は、そのページの CSS で独自に指定してあげればより直接の指示となるため共通 CSS は無視され、独自の指定が採用されます。

　基準をどのように定めるかは[こちら](#)に書いてあります。


**%**：一番使うけどその分慣れるまでやっかいなやつです。文字に対してはほぼ使わず、div や画像サイズの指定に使います。親要素を 100%として子要素のサイズを指定できると思ってください。100 以上の数値を使うことはまずないです。

　 header タグなど、親要素が一見ないものの親要素は body です。つまり、div が親でない要素に対してはブラウザの表示幅が 100％になります。


**em**：rem の親戚みたいなやつです。em は主に行間隔や文字間隔を指定するのに使っています。そのテキストタグに指定されているフォントサイズを 1em とし、比率で行間隔や文字間隔を指定できます。

　多用はしませんが、デザインにこだわるならぜひ使ってください。


　すでに TPG の公式 HP ではいろいろ検討した上での基準を設定しているので、もし単位に迷ったら誰かに聞けばいいと思います。また、例えば本文の文字サイズはまあ統一だろうということで共通読み込み用の CSS も設定してあるので、これを使うことで CSS の無駄な行も減らせる＆各ページ間で統一のとれた印象になるので、原則共通 CSS の指示を大幅に上書きすることはないほうが全体で見て綺麗だと思います。


　今回の HP 作成では共通 CSS を読み込まないので、それらの基準も今後自分で書いてもらうことになります。
