#   閑話　要素の種類について
　以前も少し書きましたが、HPを構成する要素はブロック要素、インラインブロック要素、インライン要素の3つに分けられます。


　ブロック要素は独立した領域をとり、横幅は特に指定をしない限り親要素いっぱいに広がります。pタグ、h3タグなどのテキストタグとdivがこれにあたります。そのため、`<p>テキスト1</p>`と`<p>テキスト2</p>`を連続して書いても1行ではなく2行になります。


　インライン要素はそのタグで挟まれた部分の領域のみを占有するので、独立の行を取りません。aタグとimgタグ、まだ出てきていませんがspanタグがこれにあたります。

　雑に言えばインライン要素は次の要素が隣に並んでいき、ブロック要素は行自体を占有するため次の要素は下に並ぶということです。

　実際、ヘッダーでaタグを並べていますが、これは初めから横並びになっていました。


　前のページでimgタグをわざわざブロック要素に変換したのですが、これはインライン要素は余白の設定がややこしいからです。

　まず、左右のpaddingとmarginは問題なく指定できます。上下のpaddingは、親要素であるdivのブロック領域をはみ出してつくことができてしまいます。上下のmarginはつけることができません。

　また、imgタグ以外のインライン要素は`width`、`height`の指定ができません。

　細かく覚えなくても指定ができないがちなんだなーで十分です。


　最後に、インラインブロック要素です。これは、インライン要素でできなかった`width`、`height`の指定ができるようになった要素だと思ってください。

　例えば横並びにしたいaタグが並んでいるけれども上下にmarginを取りたいときなどにブロック要素にしてしまうと要素は下に並んでいくので横並びにするという処理をする必要がありますが、インラインブロック要素にすれば横並びのままです。個人的にはあんまり使わない印象です。


　要素の種類を変更するには、CSSで`display: block ( inline / inline-block);`で任意の種類を指定します。


　要素の種類はもう一つ、中央揃えにも関わります。ややこしくなるので中央の場合のみ書きますが、centerをleftやrightに変えるだけで左右でも同じ話です。

　まず左右中央に揃えるには`text-align: center;`を指定します。これはブロック要素であれば中央に置きたい要素自体に、中央に置きたい要素がブロック要素でなければその親要素に指定します。

　次に上下中央に揃える場合ですが、上下中央ということは親要素にdivが必ずあります。なので親要素に`display: flex;`を指定して子要素を横並びにした上で使う場合がほとんどです。この場合子要素に`align-items: center;`と書きます。そしてこの場合、左右中央揃えは`justify-content`でもできます。

　親要素のdivの縦幅が定義されているなら、子要素の縦幅に対する余白を計算し`margin-top`を指定しても中央におけます。


　真面目に場合分けしてパターンをつかもうとしてもややこしいので、横並びと中央揃えのやり方を自分はこれでやると決めてしまうのがいいと思います。

　これらは別に覚える必要はなく、一度書いたコードをコピペしたりなんかいろいろあったなあと思いだして調べて使えれば十分です。
