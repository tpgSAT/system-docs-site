#   レスポンシブデザインについて
　　端末やブラウザのサイズに応じて最適な表示を自動的に適用することをレスポンシブデザインといいます。

　レスポンシブデザインの作成の手間や難易度はページの要素が煩雑になるほど上がりますが、これまでどういうdivを組みCSSを書いてきたのかにも大きく影響されます。つまりもし、ここまでで私と大きく違うコード(divの作る範囲自体が違う、など。クラス名の違いなどは関係ないです)を書いて仕様書通りの実装をしてきた人は、レスポンシブデザインのために書くコードも私と大きく異なることになります。


　レスポンシブデザインは、先ほどまで作ってきたHTMLに対し、ブラウザの幅に応じて違うCSSが適用されるようにしてつくります。

　基本的にはPC版をベースにつくり、レスポンシブ用のCSSは別で作るのがよいと思います。なのでまず、takifes_resp.cssというCSSファイルを作りましょう。ファイルの作り方は[目標物の確認・ファイル作成](/w/w01_intro/003/README.md)を参照してください。

　そしてこのCSSを使うよ、とHTMLファイルに対して宣言するため、htmlファイルの`head`タグ内にtakifes_resp.cssを読み込む行を記述してください。takifes.cssを読み込むための行をコピペして、`_resp`を付け足せば良いです。

　これで、このindex.htmlファイルはtakifes.cssとtakifes_resp.cssを読み込むことができます。


　作成したCSSの初めに文字化け防止で`@charset "utf-8";`を書きます。

　PC版表示からタブレット端末表示、スマホ表示に切り替える境目となるページ表示サイズをブレイクポイントといいます。

　TPGの公式HPでは、ブラウザの表示幅が1024px以上の時PC版、1024px以下670px以上のときタブレット端末版、670px以下の時スマホ版表示ができるようにブレイクポイントを670px、1024pxに設定しています。タブレット端末版はほぼ何も設定しなくてもさほど表示が崩れないことも多いので、まずスマホ版をつくってしまいます。


　先ほど作成したtakifes_resp.cssに次のコードを書いてください。
```css
@media screen and (max-width: 670px) {
    <!-- ここにスマホ版表示のためのCSSを記述 -->
}
```
　{  }の中には自動でインデントが起きるので、ここにこれまでと同じようにCSSで指定したい内容を記述すれば良いです。

　これは、{  }の間に書かれたCSSを適用する画面幅の最大値が670px、つまり画面幅が670px以下のときにこの中に書かれたCSSを適用するよ、ということです。


　`@media screen and (`までは半角スペースをこの通りに正確にいれてください。


　例えば、スマホ版表示ではrem単位の基準値が14pxになるので、このように書きます。
```css
@media screen and (max-width: 670px) {
    html {
        font-size: 14px;
    }
}
```
　index.htmlはtakifes.cssとtakifes_resp.cssの両方を読み込み、それぞれrem単位の基準を1.2vw、14pxと指示しているため矛盾するように思えますが、takifes.cssは画面サイズを明示していない指示なのに対して_respは画面サイズを明示した指示です。

　なのでスマホの画面に対してより直接指示をしているrespの方の指定が適用されます。一方で、takifes.cssと変更する必要のない項目に関しては改めてCSSを書く必要はありません。

　ホバーの話の時に書きましたが、変更または新たに設定する必要がある項目は書くとそちらが働き、そのままでいい指示はもとのCSSがそのまま適用されます。


　ある指示の削除はできないため、何かしらで上書きする必要があります。例えば背景色がスマホ版ではいらないとしてもPC版で設定されていては消すことができないので、`background-color: transparent;`と書くしかないということです。
