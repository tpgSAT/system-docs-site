#   ホバー処理
　ある要素にマウスをかざしたときにその要素が変化する表現をホバーといいます。例えば今作っているヘッダーのボタンは、ホバーで下線が現れるようにします。掲示物募集のボタンは色の反転です。


　ホバー処理は、要素のクラス名のあとに`:hover`をつけて指定します。つまり今回書くコードは次のようになります。
```css
.header_item:hover {
    border-bottom: 2px solid #181818;
}
```
　border-bottomは.header_itemに先ほど白色で設定しました。今回は同じ要素に#181818という色を設定しているので、ホバーしたときは#fffよりもこちらが優先され、黒色の下線がつきます。

　ホバーの時に書く項目は、通常状態に同じプロパティがあれば上書き、なければ追加されます。header_itemにはborder-bottomを設定せず、ホバーの時だけ設定しても大丈夫です。ただし、上下中央揃えをしているため、ホバーすると下線がついた分若干位置がずれることになります(実際やってみるとわかる)。

あえてそういう表現にするのもありだと思いますが、今回は位置ずれが起こらないようにしました。


　ここまでやってきたことで、「掲示物募集」のボタンが大体作れます。カラーコードや余白幅などは画像の通りなので、まずは自分でやってみてください。(※クラス名は複数付与することができます。`class="name1 name2"`のように半角スペースで区切ってください)

　ついでに、マウスをかざしたときの変化をゆっくりフェード風にしておきます。こういう細かいこだわりのためのコードは書くとキリがないので、適宜調べるのがいいと思います。(一応、TPGの公式HPにつかったコードをまとめた一覧ページもつくる予定です。)

　ホバー時の変化をゆっくりにするのはヘッダーに限らずページ全体に適用したいので、CSSで指定するセレクタとしてheader_itemではなくaにし、aタグすべてにこの処理が効くようにします。aはクラス名ではなくタグの種類なので、CSSに書く際に前に . はいりません。
```css
a {
    display: block;
    text-decoration: none;
    transition: 0.5s;<!-- 変化を0.5秒かけて行う -->
}
```
　こういう、ある種類のタグすべてに同じ処理を適用するコードは、CSSの上の方にまとめて書くことをおすすめします。上2行は、aタグはblock要素にして扱うことが多く、デフォルトの下線も消して使うことがほとんどなので一括で処理してしまうために書きました。


　header_itemには`display: inline-block;`を指定しているので矛盾した処理のように思えますが、より直接書かれている指示が優先的に適用されます。つまり、header_itemクラスはaタグの要素ですが、aタグ全体への指示よりもheader_itemを指定している指示の方が優先されるということです。

ホバー時の処理で`:hover`をつけるとつけていないときの要素が上書きされると書きましたが、それもこのためです。
