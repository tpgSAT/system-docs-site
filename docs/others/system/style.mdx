# 活動スタイル

ここでは、SAT が活動する上での心がけや、リーダーなど（もちろんリーダーに限らずみんな）に把握してほしいことを扱います。

この記述はあくまでも初代 SAT リーダーが SAT を作るときに掲げた理想像であって、代替わりなどを経てやり方や活動スタイルを改めたりしてもらっても構いません。リーダーによって理想像も違うと思うので好きにやってもらってＯＫです。


---

#### 1. なるべく時間はかけない、ただし手は抜かない

#### 2. 責務はきちんと果たす

#### 3. 最初はみんな初心者、放置せず支えよう

せっかく貴重な大学生の時間を使ってシステム班に入ってきてもらったので、なるべくタイパよく行きましょう。とは言うものの、 **タイパを意識しすぎて技術面の継承がおざなりになるのはダメです。** 
れでは システム班の責務が果たせなくなるし、TPG全体が困ることになります。
中高生の時に少しプログラミングなどをかじったことがある人とない人では理解の速さもタイピングの速さも何もかも違うので、助けが必要な人が放置されない前提ですが、できる人は速く進めてあげて、助けが必要な人にはじっくり時間をかけてあげることが重要です。
進みが速い人を助けが必要な人に合わせる必要はありません。速い人が一段階・二段階先に進むことで、速い人が速くない人を助けられるようになるメリットもあります。半年経つ頃には同学年間で完結可能な成長機会が得られるといいですね。
具体的にどのような方法で進めていくかはリーダーの裁量にかかっています。システム班の現状に合わせて柔軟に対応できるといいのかなーと思います。

#### 4. 遊びを忘れない

#### 5. メンバーの結束を大事にする

システム班のメインの活動もいいですが、たまにはご飯に行ったりお出かけしたりするのも大事です。人数が多ければ学年で分けてみたり Web とサーバーで分けてみたりいろいろ工夫できると思うので試してみましょう。
少人数型のシステム班では、メンバー同士（特に同学年メンバー）の仲が良くなる傾向にあると思うので、特に 1 ～ 2Q などは仲良くなる機会を作ってあげるといいかもしれません。
